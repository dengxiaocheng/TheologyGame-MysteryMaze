# 移动端适配计划

## 约束
- **必须纯手机浏览器可玩，禁止依赖电脑键盘（方向键、WASD、空格、回车等）**
- 所有交互必须通过触摸屏完成
- 界面布局需适配竖屏手机（375px-430px 宽度）

## 当前问题
- 移动完全依赖键盘方向键和 WASD 控制角色移动
- 没有任何触摸事件
- 主文件：`js/player.js`、`js/engine.js`、`js/ending.js`

## 修改方案

### 1. 虚拟方向键
- 在画面底部叠加一个半透明虚拟 D-Pad（上下左右四按钮）
- 样式：圆角方形按钮，60px 大小，适合拇指点击
- touchstart 开始移动，touchend 停止移动
- 映射到与 ArrowUp/Down/Left/Right 相同的逻辑

### 2. 触摸滑动移动（备选/辅助）
- 在游戏画布区域支持滑动手势
- 检测滑动方向，触发对应方向移动
- 滑动距离阈值 20px

### 3. 菜单/结尾画面
- `js/ending.js` 中的菜单导航改为点击选择，不用上下键
- 每个选项做成可点击的按钮，加上 hover/active 视觉反馈

### 4. viewport 适配
- `<meta name="viewport">` 已存在，确认 `user-scalable=no`
- canvas/game 容器宽度设为 `100vw`，高度自适应
- 防止双击缩放和长按菜单：`touch-action: manipulation`

### 5. 禁止键盘事件独占
- 保留键盘支持（兼容桌面），但键盘不再是唯一操作方式
- 移动端检测：优先显示虚拟按键；桌面端可隐藏
